# Interpretation of Hamilton Partner Choice Results

## Overview

This document summarizes insights from analyzing the line plots generated by `graphgen --study hamilton --figure s07` (partner choice under the **P** mechanism only: **no** direct or indirect reciprocity). It covers pop_1 (panels e-f, single population), pop_2 (panels a-b, two coevolving populations), and pop_3 (panels c-d, one population adapting to a fixed population). The Hamilton study uses a continuous altruism game parameterized by benefit *b* and cost *c*, with the x-axis showing *b* ŌłÆ *c*.

**Individual vs population `qBSeen`.** In the simulation output and figures, **`qBSeen` denotes the population fraction of cooperators** (C1P1 + C1P0). At the individual level, mechanism **P** does not update reputation: defectors have `qBSeen` = 0 and cooperators have `qBSeen` = 1 for their entire life. Partner choice therefore only ever reads a **binary** partner type (defector vs cooperator), not a graded image score.

Later sections emphasize **tables, correlations, and single-run statistics**; recurring mechanisms (**mutual C1P1 swaps**, **behavioral C1P0**, **sparse choosers**) are spelled out once in **Key Definitions** and not re-derived every time.

## The Game

The Hamilton altruism game is parameterized by benefit *b*, cost *c*, and given parameter *g*. With *g* = 1.0 (full dilemma), a cooperator gives all of its production *b* to its partner and pays cost *c*. The simulation parameters are:

| Parameter      | Value                         | Description                                |
| -------------- | ----------------------------- | ------------------------------------------ |
| *b* ŌłÆ *c*          | 0.008 ŌĆō 8.0 (log scale)       | Net benefit of mutual cooperation (x-axis) |
| *c*              | 1.0                           | Cost (fixed throughout; C_MIN)             |
| *b*              | *c* + (*b* ŌłÆ *c*) = 1.008ŌĆō9.0       | Benefit (varies with x-axis)               |
| *g*              | 1.0                           | Given parameter (full dilemma)             |
| *K*              | 2.0                           | Baseline fitness (k0 = k1)                 |
| *B*\_max         | 9.0                           | Maximum benefit (normalization)            |
| groupsize      | 128                           | Individuals per group from each population |

### Equivalent Prisoner's Dilemma payoffs

From `calculate_derived_globals.c`, the Hamilton game at *g* = 1.0 maps exactly to a Prisoner's Dilemma with payoffs:

| Payoff         | Formula          | Meaning                         |
| ---------------| ---------------- | ------------------------------- |
| **T** (Temptation) | 3 + (*b* ŌłÆ *c*)      | Defect while partner cooperates |
| **R** (Reward)     | 2 + (*b* ŌłÆ *c*)      | Both cooperate                  |
| **P** (Punishment) | 2                | Both defect                     |
| **S** (Sucker)     | 1                | Cooperate while partner defects |

Derivation from `calculate_derived_globals.c` with `given = 1` and symmetric populations (`b0 = b1`, `x = bŌłÆc`):
`b0 = k1 + x`, `T = k0 + b0 = k0 + k1 + x`, `R = k0 + b0 - k1 = k0 + x`, `S = k0 - k1`, `P = k0`.
For the Hamilton study constants (`k0 = 2`, `k1 = 1`), this gives `T = 3 + x`, `R = 2 + x`, `P = 2`, `S = 1`.

The key relationships are:

| Relationship | Value     | Interpretation                 |
| -------------| --------- | ------------------------------ |
| **R ŌłÆ P**        | *b* ŌłÆ *c*     | Cooperation benefit (= x-axis) |
| **T ŌłÆ R**        | 1         | Temptation premium (constant)  |
| **P ŌłÆ S**        | 1         | Sucker penalty (constant)      |

Concrete values at selected x-axis points:

| *b* ŌłÆ *c*     | T      | R      | P   | S   |
| ----------| ------ | ------ | --- | --- |
| 0.008     | 3.008  | 2.008  | 2   | 1   |
| 0.125     | 3.125  | 2.125  | 2   | 1   |
| 0.250     | 3.250  | 2.250  | 2   | 1   |
| 1.000     | 4.000  | 3.000  | 2   | 1   |
| 4.000     | 7.000  | 6.000  | 2   | 1   |
| 8.000     | 11.000 | 10.000 | 2   | 1   |

### Comparison with the PD study parameter space

In the PD study (`prisoners`), T = 1.0 and S = 0.1 are fixed while R and P vary across a 2D grid (R, P Ōłł [0.1, 1.0]). Three independent gaps vary simultaneously:

- T ŌłÆ R Ōłł [0, 0.9] ŌĆö temptation to defect from mutual cooperation
- P ŌłÆ S Ōłł [0, 0.9] ŌĆö cost of being suckered vs mutual defection
- R ŌłÆ P Ōłł [ŌłÆ0.9, 0.9] ŌĆö benefit of mutual cooperation vs mutual defection

In Hamilton, only **R ŌłÆ P varies** (= *b* ŌłÆ *c*), while T ŌłÆ R = P ŌłÆ S = 1 are constant. This means:

1. **The Hamilton game has a fixed temptation premium** (T ŌłÆ R = 1) compared to the PD's variable temptation (max 0.9). A defector always gains 1 fitness unit by exploiting a cooperator, regardless of *b* ŌłÆ *c*.
2. **The sucker penalty is fixed** (P ŌłÆ S = 1). A cooperator paired with a defector gets fitness 1, which is 1 below mutual defection.
3. **R ŌłÆ P spans a much wider range** (0.008 to 8.0 vs PD's ŌłÆ0.9 to 0.9). At high *b* ŌłÆ *c*, mutual cooperation is enormously more profitable than mutual defection.

Fitness is normalized as (*K* + *w*) / (*K* + *B*\_max) = (*K* + *w*) / 11. Without any cooperation mechanisms, the theoretical prediction at *g* = 1.0 is zero cooperation: the condition (1ŌłÆ*g*)┬Ę*b* > *c* is never satisfied. Partner choice must bootstrap cooperation from scratch.

## Key Definitions

| Term   | Definition                                                            |
| ------ | --------------------------------------------------------------------- |
| C1P1   | Cooperators who choose partners ("Choosers")                          |
| C0P1   | Defectors carrying P1 (P1 is **silent** ŌĆö they don't actually choose)     |
| C1P0   | Cooperators who don't choose partners ("non-chooser cooperators"); **free-riders** in the behavioral sense below |
| P1     | Total frequency of P1 allele = C0P1 + C1P1                            |
| qBSeen | **In tables and figures:** population cooperation rate (= C1P1 + C1P0). **Per individual (P only):** fixed 1 if C1, 0 if C0 ŌĆö no reciprocity, so it never drifts between rounds. |
| AllC   | Frequency of C1P0 (cooperators without partner choice)                |
| w╠ä      | Normalized mean fitness                                               |

**C1P0 as free-riders.** Throughout this document, *free-rider* for C1P0 is **behavioral**, not ŌĆ£cooperation without paying a fitness cost.ŌĆØ C1P0 **do not observe othersŌĆÖ behavior for partner sorting** (they never invoke partner choice), **do not discriminate non-cooperators** in that sense, and **do not take part in mutual swaps**ŌĆöonly C1P1 enter the swap pool under `choose_partner.c`. They still benefit when C1P1 maintain a cooperative partner pool.

From `choose_partner.c` (`choose_partner_L0P1`), a chooser is **eligible** to upgrade when:
```c
ind->Choose != 0 && ind->qBSeen != 0 && ind->partner->qBSeen == 0
```
Under **P** alone, C1P1 individuals leave a C0 partner for a C1P1. The two C1P1 upgrade their fitness from S to R. The two C0 end up paired with each other and downgrade their fitness from T to P. C0P1 individuals carry P1 but never activate it; C1P0 are absent from this pool.

### C1P0 share rises at high *b* ŌłÆ *c* (choosers can be few but are not optional)

Conceptually: partner choice still underwrites the pool; at high *b* ŌłÆ *c* a **thin C1P1 layer** can support many **C1P0** (sparse choosers ŌĆö Key Definitions). Quantitatively: C1P0/qBSeen rises from **9%** (*b* ŌłÆ *c* = 0.25) to **33%** (*b* ŌłÆ *c* = 8.0); **P1** allele frequency falls because **P0** spreads among cooperators, not because sorting switches off.

**Single-run (`hamilton_1run`, pop_1, `shuffle_cost12_128`, P, given = 1.0, 9 timesteps/cell):** **C1P1** and **C1P0** move opposite in most adjacent pairs; with **qBSeen** ~flat at high *b* ŌłÆ *c*, anti-phase is **mostly** **C1P1 + C1P0 = qBSeen**, plus **P0**/**P1** turnover among cooperators.

**GroupsizeŌĆ»4 vs 128 (`hamilton_1run`, same 21 cells, movie `csv_0_for_movie.con`, pop_1):** **`shuffle_cost12_4`** yields **narrow, nearly flat `qBSeen` trajectories** across the nine snapshotsŌĆötypical within-cell range **~0.07** vs **~0.67** at **`shuffle_cost12_128`**, with **large last-interval moves rare at 4 (~5% of cells vs ~62% at 128)**ŌĆöso **groupsizeŌĆ»4** matches **low end-state cooperation** as a **different regime**, not **128 slowed down**.

**Invasion vs late tail (partition timesteps within each cell):** **Invasion** (first three ╬ö after *t* = 1): **r**(C0, ╬öP1) Ōēł **+0.71**, **r**(C0, ╬öC0) Ōēł **ŌłÆ0.62**; high C0 at *t* pairs with large positive **╬öP1** and negative **╬öC0**. **Late** (*t* index Ōēź 3): **r**(C0, ╬öP1) Ōēł **+0.08** ŌĆö tiny stepwise ╬öŌĆÖs; **no** reliable one-step **P1** Ōåö **P0** cycle at nine snapshots (longer period, smaller amplitude, or both). Sharper tail inference: **many replicates** at fixed *b* ŌłÆ *c* and/or **dense checkpoints only near run end** (limits CSV growth vs uniform dense output). High **C1P0**/qBSeen at *t* ŌåÆ slightly positive mean next-step ╬ö(C1P1/qBSeen) (**suggestive**).

**P1 down at high *b* ŌłÆ *c***: dilution by **C1P0**, not loss of partner choice as the sustaining process.

## P1 Hitchhiking

### The pattern

P1 frequency peaks at a *lower* *b* ŌłÆ *c* than C1P1 frequency. This occurs in all three population structures:

| Population       | P1 max *b* ŌłÆ *c*     | P1    | C1P1 max *b* ŌłÆ *c*     | C1P1  |
| ---------------- | ---------------- | ----- | ------------------ | ----- |
| pop_1            | 0.25             | 0.887 | 0.50               | 0.772 |
| pop_2 (fset_0)   | 2.00             | 0.893 | 5.66               | 0.769 |
| pop_3 (evolving) | 2.00             | 0.858 | 4.00               | 0.736 |

### The mechanism

This is the same neutral hitchhiking observed in the Prisoner's Dilemma analysis. The P1 allele is phenotypically silent in defectors: C0P1 individuals carry P1 but never activate partner choice. The P1 maximum occurs where:

1. C1P1 is still high (partner choice is still under positive selection among cooperators)
2. Enough defectors persist (C0 is present to serve as neutral P1 carriers)
3. Mutation from C1P1 ŌåÆ C0P1 creates a reservoir of neutral P1 carriers

At the P1 maximum in pop_1 (*b* ŌłÆ *c* = 0.25):
- C1P1 = 0.727 (high ŌĆö cooperation is well-established)
- C0P1 = 0.160 (substantial ŌĆö 16% of the population are defectors carrying P1)
- P1 = 0.887

At the C1P1 maximum (*b* ŌłÆ *c* = 0.50):
- C1P1 = 0.772 (highest)
- C0P1 = 0.082 (low ŌĆö defection is suppressed)
- P1 = 0.854 (lower than at *b* ŌłÆ *c* = 0.25)

As *b* ŌłÆ *c* increases from the P1 max:
- C1P1 increases slowly (0.727 ŌåÆ 0.772, ╬ö = +0.044)
- C0P1 decreases faster (0.160 ŌåÆ 0.082, ╬ö = -0.078)

The net effect: P1 = C1P1 + C0P1 **decreases** because the loss of neutral carriers outpaces the gain in selected carriers.

### Why the P1 max shifts across population structures

In pop_1, the P1 maximum occurs at *b* ŌłÆ *c* = 0.25 ŌĆö very early in the *b* ŌłÆ *c* range, because cooperation takes hold at low *b* ŌłÆ *c* in single populations. In pop_2 and pop_3, the P1 maximum shifts to *b* ŌłÆ *c* Ōēł 2.0 because cooperation requires higher *b* ŌłÆ *c* to establish in between-population pairing. The P1 peak always occurs in the transition zone where C1P1 is abundant but defection has not yet been fully suppressed.

### The exploitation pattern

At *b* ŌłÆ *c* = 8.0:
- The cooperative population (fset_0) has qBSeen = 0.926 but fitness = 0.259
- The defecting population (fset_1) has qBSeen = 0.197 but fitness = 0.922
- The defecting population has **3.56├Ś higher fitness**

The cooperative population pays costs to benefit the defecting population's individuals, who receive benefits without paying. Partner choice hits the **same bottleneck** as elsewhere (Key Definitions): swaps need **counterpart C1P1**; the defecting population contributes few **choosers**, so mismatches persist.

In unnormalized terms at *b* ŌłÆ *c* = 8.0:
- fset_0 fitness: 0.259 ├Ś 11 = 2.84 (barely above baseline *k* = 2)
- fset_1 fitness: 0.922 ├Ś 11 = 10.14 (near maximum *k* + *B*\_max = 11)

### The asymmetry onset

At low *b* ŌłÆ *c* (< 0.125), both populations are symmetric: mostly defectors with ~4ŌĆō5% cooperation. The asymmetry grows continuously and is already pronounced by *b* ŌłÆ *c* = 0.5 (╬öqBSeen = 0.151). There is no sharp threshold ŌĆö unlike the Prisoner's Dilemma where cooperation shows a bimodal phase transition near R Ōēł P. Instead, the Hamilton game produces a **gradual** symmetry breaking as *b* ŌłÆ *c* increases.

Single-run data (`hamilton_1run`) confirms that this gradual transition is **genuine, not an averaging artifact**: each *b* ŌłÆ *c* value shows a stable intermediate cooperation level that persists across all 9 timesteps (std typically 2ŌĆō8%), not all-or-nothing jumps that average into a gradient. The cooperation rises smoothly from ~0.04 at low *b* ŌłÆ *c* through ~0.34 at *b* ŌłÆ *c* = 0.18 to 0.58ŌĆō0.85 at high *b* ŌłÆ *c*. Hamilton's constant T ŌłÆ R = 1 keeps a persistent temptation to defect at every parameter value, unlike the PD where varying T ŌłÆ R can approach zero and create sharper bistability boundaries.

### The paradox of success

The more successful cooperation is in one population, the more exploitable that population becomes. The correlation between "more choosers" and "lower fitness" is ŌłÆ0.92, matching the pattern found in the Prisoner's Dilemma (where the correlation was ŌłÆ1.00).

Single-run data confirms that the asymmetry is an **absorbing state** at high *b* ŌłÆ *c*: zero role flips occur across all 9 timesteps. Which population becomes the cooperator is **stochastic** ŌĆö at *b* ŌłÆ *c* = 2.0 and 4.0, fset_0 is the cooperator, but at *b* ŌłÆ *c* = 2.83, 5.66, and 8.0, fset_1 is the cooperator instead. Each cell's symmetry breaking is an independent stochastic event; the averaged data always labels fset_0 as the higher-qBSeen population because it sorts after averaging. At low *b* ŌłÆ *c*, roles flip frequently (2ŌĆō5 times per cell), consistent with cooperation being too weak to lock in. The weaker averaged correlation in Hamilton (ŌłÆ0.92 vs PD's ŌłÆ1.00) likely reflects this low-*b* ŌłÆ *c* noise rather than role switching at high *b* ŌłÆ *c*.

## Evolving vs Fixed Population (pop_3, panels c-d)

### Sharp transition in the evolving population

In pop_3, the evolving population adapts to a fixed population with 25% each of C0P0, C0P1, C1P0, C1P1 (qBSeen = 0.50, P1 = 0.50 constant). The evolving population shows a sharp cooperation transition:

| *b* ŌłÆ *c*     | qBSeen (evolving) | P1        | C1P1      |
| --------- | ----------------- | --------- | --------- |
| 0.250     | 0.047             | 0.517     | 0.034     |
| 0.354     | 0.053             | 0.538     | 0.040     |
| 0.500     | 0.071             | 0.526     | 0.056     |
| **0.707**     | **0.408**             | **0.746**     | **0.383**     |
| 1.000     | 0.487             | 0.782     | 0.459     |
| 1.414     | 0.585             | 0.827     | 0.545     |
| 2.000     | 0.702             | 0.858     | 0.641     |

Between *b* ŌłÆ *c* = 0.5 and *b* ŌłÆ *c* = 0.707, cooperation jumps from 7.1% to 40.8% ŌĆö a **~6├Ś increase** in one step. This is the same positive-feedback tipping point seen in the Prisoner's Dilemma: once enough **C1P1** exist for mutual swaps, partner choice activates ŌåÆ choosers pair with other choosers (*R* where swaps succeed) ŌåÆ higher fitness ŌåÆ more cooperators.

Single-run data (`hamilton_1run`) confirms this is a **sharp one-time transition**: cooperation jumps at the first measured timestep (t = 131072) and remains stable throughout (std Ōēł 0.02). No cycling is observed ŌĆö the positive-feedback cascade ignites once and persists, identical to the pattern in `prisoners_1run` pop_3.

### Why the transition is sharper than in pop_1

In pop_1, cooperation is 13.2% even at *b* ŌłÆ *c* = 0.008, and rises smoothly. In pop_3, cooperation stays below 7% until *b* ŌłÆ *c* Ōēł 0.7, then jumps. The difference is the pairing structure:

- **pop_1** (within-population): Both partners at an interaction come from the **same** population (still well-mixedŌĆöno spatial layout). Mutual swaps only need another **C1P1** from **that** pool, not coordinated rare types across two populations. A rare C1P1 paired with a defector can mutually swap with another such C1P1 and immediately gain a C1P1 partner. Partner choice works at any cooperator frequency. **This is pairing/matching structure (one pool of interaction partners), not ŌĆ£two C1P1 are likelier because they share ancestry.ŌĆØ** Finite-population relatedness is secondary here.
- **pop_3** (between-population): The evolving population pairs only with the fixed population. For a mutual swap, a C1P1 in the evolving population stuck with a defector in the fixed population needs **another C1P1** in the evolving population who is also stuck with a defector from the fixed population (`choose_partner_L0P2` pairs choosers across populations the same way). When cooperators are rare in the evolving population, such pairs are unlikely. Cooperation must cross a threshold density before the swap cascade ignites.

The direction of exploitation reverses as cooperation increases in the evolving population:

## Shuffle Effect

Shuffling partners randomly within groups every time step has **negligible effect** on cooperation, P1, or fitness in the Hamilton study. Across all three population structures and the full *b* ŌłÆ *c* range, the difference between shuffle and no-shuffle conditions is typically < 0.02 in qBSeen and < 0.01 in P1.

Shuffle matters little: the next step re-applies the same swap logic (Key Definitions); *b*ŌĆō*c* payoffs make the outcome insensitive to whether pairs were broken up.

## Comparison with Prisoner's Dilemma

Both studies share T > R > P > S and `choose_partner.c` (Hamilton = mechanism **P** only; binary `qBSeen` ŌĆö Overview). Hamilton fixes **T ŌłÆ R = P ŌłÆ S = 1** and varies only **R ŌłÆ P = *b* ŌłÆ *c***.

### Same phenomena

1. **P1 hitchhiking** ŌĆö see **P1 Hitchhiking**; PD mechanism identical.
2. **pop_2 asymmetry / exploitation** ŌĆö ╬öqBSeen vs ╬öfitness **ŌłÆ0.92** (PD **ŌłÆ1.00**); **Two Coevolving Populations**.
3. **pop_3 crossover** ŌĆö **Evolving vs Fixed Population**.
4. **Swap bottleneck** ŌĆö Key Definitions (**C1P0** cannot swap; need **counterpart C1P1**).

### Different phenomena

1. **pop_2 transition shape**: PD is bimodal near **R Ōēł P** (variable **T ŌłÆ R**); HamiltonŌĆÖs **T ŌłÆ R = 1** yields **gradual** symmetry breaking ŌĆö single-run: stable intermediate **qBSeen** at each *b* ŌłÆ *c* (std ~2ŌĆō8%), not a blended artifact.
2. **pop_1 floor**: PD **qBSeen** Ōēł 0.49 at lowest **R ŌłÆ P** (0.041); Hamilton **0.13** at 0.008 ŌĆö HamiltonŌĆÖs fixed temptation (**T ŌłÆ R** = 1) vs PDŌĆÖs **T ŌłÆ R** ŌåÆ 0 at low *R*.
3. **High R ŌłÆ P**: C1P0 share **~33%** of cooperators at **R ŌłÆ P = 8** vs **9%** at 0.25 ŌĆö sparse choosers (**pop_1** table); **R/P = 5** at *b* ŌłÆ *c* = 8 (payoffs in **The Game**).
4. **Exploitation ratio at R ŌłÆ P = 8**: **3.56├Ś** fitness (Hamilton **T** up to **11** vs PDŌĆÖs compressed scale).

## Given = 0.5: Correct Game-Type Classification

Using the exact Hamilton branch equations from `calculate_derived_globals.c` (`given < 1.5`), with `k0 = 2`, `k1 = 1`, and `x = b - c`:

- `b = k1 + x = 1 + x`
- `T = k0 + g*b = 2 + g(1 + x)`
- `R = k0 + b - k1 = 2 + x`
- `P = k0 = 2`
- `S = k0 + (1-g)b - k1 = 1 + (1-g)(1 + x)`

At `g = 0.5`, this becomes:

- `T = 2.5 + 0.5x`
- `R = 2 + x`
- `P = 2`
- `S = 1.5 + 0.5x`

So `T - R = P - S = 0.5(1 - x)`, giving a clean regime split at `x = 1`:

- `x < 1`: `T > R > P > S` (PD)
- `x = 1`: `T = R > P = S` (boundary)
- `x > 1`: `R > T > S > P` (harmony-like)
